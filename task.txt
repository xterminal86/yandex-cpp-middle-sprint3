Практическая работа: ревьювер
Вы изучили различные контейнеры современной стандартной библиотеки С++ и 
алгоритмы для работы с ними. Чтобы применить полученные знания на практике, 
предлагаем выполнить проектную работу, в которой вам предстоит:
Разработать быстрый и эффективный шаблонный класс-контейнер для работы с книгами 
BookDatabase, в том числе:
концепты и стандартные предикаты;
объявление псевдонимов типов для совместимости со стандартными алгоритмами;
хранение данных с помощью std::unordered_set,std::vector, std::flat_map, 
std::flat_set и гетерогенного поиска.
Разработать несколько алгоритмов для получения информации о коллекции книг, 
например, среднего рейтинга книг по жанрам.
Реализовать аналоги std::all_of для формирования коллекции книг по заданным 
условиям с помощью свёрточных выражений (fold expressions). 
Fold expressions — это инструкция по развёртыванию пачки шаблонных параметров 
(variadic template pack), согласно которой компилятор применяет операцию к 
каждому элементу пачки. Подробнее.

В результате вы разработаете собственный контейнер с быстрым и удобным 
интерфейсом для работы с коллекцией элементов, а ещё вспомогательные функции и 
алгоритмы для работы с ним. В процессе вы будете применять концепты, свёрточные 
выражения и новые плоские контейнеры.

Разработайте методы анализа текущего состояния контейнера в файле statistics.hpp:
 
Функция buildAuthorHistogramFlat должна создавать гистограмму количества книг по 
авторам, используя один из доступных flat-контейнеров.
Функция calculateGenreRatings должна возвращать средний рейтинг книг по жанрам, 
используя один или несколько из доступных flat-контейнеров.
Функция calculateAverageRating должна использовать стандартные алгоритмы для 
вычисления среднего рейтинга всех книг в библиотеке.
Функция sampleRandomBooks должна выбирать указанное количество книг из библиотеки 
и возвращать их в виде std::vector<std::reference_wrapper<const Book>>.
Функция getTopNBy должна выбирать из библиотеки указанное количество книг c 
наивысшим рейтингом и возвращать их в виде 
std::vector<std::reference_wrapper<const Book>>. Это единственная функция, 
которой разрешено изменять переданный контейнер.
Все функции, за исключением последней, не должны модифицировать входные 
параметры.
Сигнатуры для перечисленных методов должны выглядеть так:

  template <BookContainerLike T, typename Comparator = TransparentStringLess>
  auto buildAuthorHistogramFlat(const BookDatabase<T> &cont, Comparator comp = {});
  
  template <BookIterator It, BookSentinel<It> Sent>
  auto calculateGenreRatings(It first, Sent last);
  
  template <BookContainerLike T>
  double calculateAverageRating(const BookDatabase<T> &books);
  
  template <BookContainerLike T>
  auto sampleRandomBooks(const BookDatabase<T> &cont, size_t num);
  
  template <BookContainerLike T, typename Comparator>
  auto getTopNBy(BookDatabase<T> &cont, size_t n, Comparator comp); 
Разработайте функции-помощники для быстрого и эффективного написания фильтров 
для контейнера BookDatabase:
YearBetween / RatingAbove / GenreIs и all_of / any_of должны быть реализованы с 
использованием лямбда-функций и являться фабриками по созданию унарных 
предикатов для элементов типа Book.
all_of / any_of должны принимать переменное число унарных предикатов.
Алгоритм filterBooks должен принимать пару итераторов на контейнер BookDatabase 
и возвращать std::vector<std::reference_wrapper>> на элементы из заданного 
диапазона.
Сигнатуры для перечисленных методов должны выглядеть так:

  constexpr auto YearBetween(int start, int end);
  constexpr auto RatingAbove(double min_rating);
  constexpr auto GenreIs(Genre genre);
  
  template <BookIterator It, BookSentinel<It> Sent, BookPredicate Pred>
  auto filterBooks(It first, Sent last, Pred pred);
  
  template <BookPredicate... Preds>
  constexpr auto all_of(Preds... preds);
  
  template <BookPredicate... Preds>
  constexpr auto any_of(Preds... preds);
